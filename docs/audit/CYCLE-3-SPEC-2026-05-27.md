## 1. Architecture overview — what we're building

Three screens collapse into ONE workspace (matching the Codex mockup):

```
┌─────────────────────────────────────────────────────────────────────┐
│  TopBar: Account · Bal · Eq · Free Margin · Margin Level · Open Positions │
├─────────────┬──────────────────────────────────────┬─────────────────┤
│  LeftRail   │  Center (workspace)                   │  RightRail     │
│  ─────────  │  ───────────────────                  │  ────────────  │
│  Dashboard  │  • Market Watch (top half)             │  AI Dock       │
│  Markets    │  • Open Positions (mid)                │  • Real/Sim    │
│  Strategy   │  • Pending Orders (bottom)             │  • Activity    │
│  Positions  │  + ASP overlay when symbol selected    │  • Proposed    │
│  Orders     │  + Chart overlay when chart selected   │    Action      │
│  News       │  + Strategy pipeline when Strategy tab │  • Confirm/    │
│  Settings   │                                        │    Reject      │
└─────────────┴──────────────────────────────────────┴─────────────────┘
│  BottomRibbon: Compact Market Watch + Open Positions tally (always-on)│
└─────────────────────────────────────────────────────────────────────┘
```

**Key UX principles** (mirroring cTrader Pro / Codex mockup):
1. **Account context never leaves the screen** — TopBar + BottomRibbon together hold balance/equity/margin/open visible from every tab.
2. **AI Dock is always-on right rail**, not a tab — it surfaces `proposed_action` candidates (close, modify SL/TP, open new) with rationale; operator approves or rejects from there.
3. **Information density 5-10× current** — proper data tables (not chips), with sortable columns and right-click context menus.
4. **Single workspace, not separate tabs** — Market Watch, Positions, Orders all visible together; chart and ASP open as overlays on demand.

### 2.1 Market Watch panel

**Reference:** cTrader Help — [Market Watch](https://help.ctrader.com/ctrader/interface/market-watch/), [MarketWatch: Symbols and Watchlists](https://help.ctrader.com/ctrader-web/interface/marketwatch).

**Columns (visible by default):**

| Column | cTrader name | Source field | Update rate | Notes |
|---|---|---|---|---|
| Symbol | Symbol | `ProtoOALightSymbol.symbolName` | static | + open positions badge (count) inline |
| Bid | Bid | `ProtoOASpotEvent.bid` (×10^-digits) | <2s tick | colored red on tick-down |
| Ask | Ask | `ProtoOASpotEvent.ask` (×10^-digits) | <2s tick | colored green on tick-up |
| Spread | Spread | computed `ask - bid` in pips | <2s tick | wide-spread highlight (red) when > avg + 2σ |
| Δ% | Daily change | `(ask - prev_day_close) / prev_day_close * 100` | every 30s | red/green coded, magnitude formatted |
| High | Daily high | computed from M1 candles over last 24h | 1m | sparkline cell |
| Low | Daily low | computed from M1 candles over last 24h | 1m | |
| Sentiment | Market sentiment | `ProtoOAGetTickDataReq` aggregated buys vs sells (or external feed) | every 60s | bar chart cell, percent buy |

**Columns (hidden by default, opt-in):** Open Time UTC, Tradeable (mode/short), Lot Size (cents), Commission, Long Swap, Short Swap, Margin Required.

**Filters / watchlists:**
- Tab strip: All / Forex / Metals / Crypto / Equities / Indices / Custom
- "Custom" = saved per-user watchlists (e.g. "Majors", "Carry trades", "London session")
- Search bar with typeahead (current: works)

**Row interactions:**
- Single-click row → opens chart in center workspace
- Double-click → adds to Active Symbol Panel for full order ticket
- Right-click → context menu: Open chart, Add to watchlist, Set price alert, Buy market, Sell market

**Color rules:**
- Bid/Ask flash green on uptick / red on downtick for 200ms
- Δ% colored by sign, intensity by magnitude (bucket >2% loud)
- Spread > 2σ over symbol-history highlighted in `NeoethosTokens.warning`

**Backend endpoints needed (new):**
- `GET /broker/symbols/extended` — full ProtoOASymbol per symbol cached (not just name+enabled)
- `GET /broker/watchlists` + `POST /broker/watchlists` — user-defined groupings
- `GET /broker/sentiment?symbols=...` — per-symbol sentiment %
- `GET /chart/aggregated_stats?symbol=X&window=1d` — High/Low/PrevClose for Δ% calc

**Files to create/modify:**
- New: `experiments/forex-flutter-ui/lib/screens/market_watch_screen.dart`
- New: `experiments/forex-flutter-ui/lib/widgets/market_watch_row.dart`
- Modify: `crates/neoethos-app/src/server/symbols.rs` (extended catalog DTO)
- Modify: `crates/neoethos-app/src/app_services/ctrader_data.rs` (parse all 40 ProtoOASymbol fields, not 5)

**Effort:** 12h backend + 14h Flutter = ~26h. Foundation for everything else.

### 2.4 AI Dock (right rail, always-on)

**This is the most novel piece of the Codex mockup** — no equivalent in cTrader Pro/MT5. Inspired by TradingView Copilot pattern.

**Sections (top to bottom):**

**A. Mode toggle:** Real / Physic / Simulation (3-way)
- Real = production trading, broker-bound
- Physic = read-only mirror (auto-trade off, manual on)
- Simulation = paper trading (separate journal, no broker send)

**B. Data sources card:** lists what AI is currently using
- ✓ Live spot stream (8 symbols subscribed)
- ✓ Trades (last 30 days)
- ✓ Indicators (SMA/EMA/RSI active on EURUSD M1)
- ✓ News feed (ForexFactory + 2 RSS sources)
- Disable individual sources to test ablation

**C. Recent Activity log:** scrollable chronological list
- "13:42 — Discovered 73 candidates, kept top 3"
- "13:51 — Order #304146278 SELL 0.01 EURUSD filled at 1.16233"
- "14:02 — Net unrealized PnL crossed -£20, sent warning"

**D. Proposed Action card (the big feature):**
- Shows top-1 LLM-proposed action with full rationale
- E.g., "Close 0.25 lots EURUSD"
  - "Confidence: 78%"
  - "Reasoning: RSI(M5)=78 overbought, NFP in 12min, current P&L +12 pips."
  - "Expected outcome: lock in +$12 before news volatility."
- Toggle: "DEMO AUTO-EXECUTE Enabled" (when ON, action fires after N-sec countdown unless rejected)
- Buttons: **Review & Confirm** / **Reject** / **Modify**

**E. Confirmation flow:**
- "Review & Confirm" opens a modal with:
  - Full order parameters editable
  - Expected slippage band (from spread × √volume)
  - Risk check (max DD %, position size %)
  - "Send" sends to broker; result lands in Activity log

**Backend endpoints:**
- `GET /actions/pending` (exists, mostly) — extend with rationale + confidence
- `POST /actions/{id}/approve` — operator approval, hits broker
- `POST /actions/{id}/reject` — operator rejection, logs reason
- `POST /actions/{id}/modify` — operator edits before approving
- `GET /ai/data-sources` — list active sources
- `POST /ai/data-sources/{name}/toggle` — enable/disable

**Files:**
- New: `experiments/forex-flutter-ui/lib/widgets/ai_dock.dart` + sub-widgets
- New: `crates/neoethos-app/src/app_services/proposed_actions.rs` (LLM → action conversion)
- Modify: `crates/neoethos-app/src/server/pending_actions.rs` (add rationale field)

**Effort:** 12h backend + 16h Flutter = ~28h. Mid-difficulty.

### 2.5 Strategy Lab — unified pipeline view (Image 2)

**Replaces:** current 3 separate screens (Discovery / Training / Intelligence).

**Layout:** Horizontal stepper showing pipeline stages with green-check / progress-bar / red-X per stage:

```
Data Ready ── Discovery ── Training ── Test ── Validation ── Promotion Gate
   ✓             ✓           ⏳         ⃝          ⃝              ⃝
```

**Per-stage panel (clickable, expands inline):**

**Data Ready:**
- Data dir + symbols loaded
- Last touch timestamp
- "Bootstrap data" button (current Data Bootstrap screen functionality)

**Discovery:**
- GA hyperparameters editable (population, generations, mutation rate — already exposed per #194)
- Per-candidate trade histogram (real-time during run)
- FunnelProfile output: how many candidates passed each of the 16 stages
- "Start" / "Stop" / "Pause" buttons

**Training:**
- Selected discovery output candidates
- Walk-forward windows configured
- Per-model performance summary
- Promote/discard buttons per candidate

**Test:**
- Out-of-sample test results
- Sharpe / max DD / win rate / profit factor

**Validation:**
- Monte Carlo simulations
- Sensitivity to spread / commission / slippage
- Correlation analysis vs existing strategies

**Promotion Gate:**
- Final accept/reject decision
- "Promote to live" button — copies model to `models/active/`
- Audit trail of why each gate decision was made

**Backend:**
- New: `GET /strategy/pipeline/{id}` — returns state of all 6 stages for current pipeline run
- Existing `/engines/status` extends with FunnelProfile + per-stage detail

**Effort:** 18h Flutter + 6h backend = ~24h. Most-impact UX change for the "Search → Trade" workflow.

## 3. Backend foundation (must happen before UI rebuild)

These are the audit findings that BLOCK the UI redesign — the new UI can't display data we're not fetching. **Order:**

### Phase A — Symbol catalog (closes D.1, D.2, D.3, D.4, D.5, D.7 from batch 2)

**File:** `crates/neoethos-app/src/app_services/ctrader_data.rs`

Today reads 5 fields of `ProtoOASymbol`. Extend to read **all financial fields:**

```rust
pub struct SymbolFull {
    // identity
    pub symbol_id: i64,
    pub symbol_name: String,
    pub description: String,
    // pricing / volume
    pub digits: i32,
    pub pip_position: i32,
    pub lot_size_cents: i64,          // already there
    pub min_volume: i64,              // already there
    pub max_volume: i64,              // already there
    pub step_volume: i64,             // already there
    // commission (NEW)
    pub commission_type: CommissionType,         // USD_PER_MILLION_USD | USD_PER_LOT | PERCENTAGE_OF_VALUE | QUOTE_CCY_PER_LOT
    pub precise_trading_commission_rate: i64,    // ×10^8
    pub precise_min_commission: i64,             // ×10^8
    pub min_commission_asset: String,            // USD default
    pub min_commission_type: MinCommissionType,
    pub pnl_conversion_fee_rate: i32,            // 1=0.01%, applies if quote != deposit
    // swap (NEW)
    pub swap_long: f64,
    pub swap_short: f64,
    pub swap_calculation_type: SwapCalculationType, // PIPS | PERCENTAGE | POINTS
    pub swap_period_hours: i32,                  // 24=daily, etc.
    pub swap_time_minutes: i32,                  // minutes from 00:00 UTC
    pub swap_rollover_3_days: DayOfWeek,         // triple-swap day
    pub charge_swap_at_weekends: bool,
    pub skip_swap_periods: i32,
    // distance constraints (NEW)
    pub sl_distance_points: u32,                 // min SL distance
    pub tp_distance_points: u32,                 // min TP distance
    pub gsl_distance_points: u32,                // min guaranteed-SL distance
    pub distance_set_in: SymbolDistanceType,     // POINTS | PERCENTAGE
    // trading mode (NEW)
    pub trading_mode: TradingMode,               // ENABLED | DISABLED_WITHOUT_PENDINGS | etc.
    pub enable_short_selling: bool,
    // schedule
    pub trading_intervals: Vec<TradingInterval>,
    pub schedule_timezone: String,
    pub holidays: Vec<Holiday>,
    // misc
    pub max_exposure: u64,
    pub measurement_units: String,
}
```

**Plumbing:**
1. Fetch via `ProtoOASymbolByIdReq` (already wired, just unused fields)
2. Cache per-account in `app_services::broker_persistence.rs`
3. Expose via new `GET /broker/symbols/extended` endpoint
4. Backtest cost model (`crates/neoethos-search/src/cubecl_eval.rs`) reads per-symbol commission + swap from this cache

**Effort:** ~10h. **HIGHEST LEVERAGE FIX IN THE CODEBASE.**

### Phase B — Volume conversion final verification

Already fixed in commits 6cd24a78 + 2d0bd8ec but **needs end-to-end test:** place 0.01 lot, verify broker reports 100,000 wire, snapshot shows volume:1,000, close sends 100,000 wire and succeeds. Add to `discovery_tests.rs` and `ctrader_execution_tests.rs`.

**Effort:** 2h.

### Phase C — Pre-trade gates wiring

In `crates/neoethos-app/src/app_services/trading/risk_gate.rs`:
1. **Volume bounds:** check `volume_units` against symbol's `min_volume` and `max_volume`. Reject with clear message.
2. **SL distance:** check `relative_stop_loss >= sl_distance_points × pip_size`. Reject if too close.
3. **TP distance:** same.
4. **Trading mode:** reject if symbol in CLOSE_ONLY_MODE for new positions.
5. **Short-selling:** reject SELL on symbols where `enable_short_selling = false`.

All five rejections must produce a unique error code that maps to a clear Flutter error translation (extend `ctrader_errors.rs`).

**Effort:** 8h.

### Phase D — Backtest cost model rewrite

In `crates/neoethos-search/src/cubecl_eval.rs` + scoring code:
1. Replace global `commission_per_trade` with per-symbol calculation:
   - `commission = volume_in_lots × precise_trading_commission_rate × {1e-8 for PER_LOT, ...}`
   - `min_commission` clamp
2. Add overnight swap charge:
   - For each backtest day where position open at swap_time, accrue `swap_long` or `swap_short`
   - Triple charge on `swap_rollover_3_days`
3. Add `pnl_conversion_fee_rate` when quote ≠ deposit currency

**Effort:** 12h (the math is intricate, tests are crucial).

### Phase E — A.3 PnL pips formula (bridge.rs)

The pip-conversion math currently mixes account-ccy PnL with quote-ccy pip value. Documented in batch 1 finding A.3. Fix:
```rust
// pips = price_movement / pip_size  (account-ccy independent)
```

Existing override at bridge.rs:619-656 already does this correctly for live tick. Make the broker-derived path use the same logic.

**Effort:** 3h + 2h to fix the misleading test fixture (E.1 in batch 2).

### Phase F — Codex Responses model whitelist resolution

Two paths:
1. **Investigation:** capture official Codex CLI HTTP traffic via mitmproxy or `--debug` flag to discover required headers (`x-codex-installation-id` etc.). Add to client.rs. Test.
2. **Fallback:** if (1) is blocked, switch AI Helper to `api.openai.com/v1/responses` with an operator-provided API key (breaks "no API key" promise but unblocks chat). Add Settings field for the key.

**Effort:** 4h investigation + 4h either fix.

## 6. The honest case for Cycle 3

> Owner: "Είναι δεκάδες πραγματικά δεκάδες τα προβλήματα... αν κάνεις τόσα λάθοι σίγουρα θα σας γυρίσουμε την πλάτη!"

The previous 2 cycles built infrastructure but **read cTrader as a 5-field key-value store**, missing the 35+ fields that determine real PnL. The result is a system that:
- looks profitable in backtest but loses live (no swap, no per-symbol commission)
- rejects 0.01-lot orders at the broker (volume math wrong, now fixed but tested-with-wrong-fixture)
- shows information across 8 screens that pros expect on 1
- has an AI assistant that proxies dumb chat instead of proposing actions with rationale

Cycle 3 closes the loop. 5 weeks of focused work, in the order above, produces a system that a real trader can use without first reading 47 audit docs to understand which numbers are wrong.

After Cycle 3 the comparison should be:
- Cycle 2 NeoEthos: "looks like a trading app, math is a minefield"
- Cycle 3 NeoEthos: "looks like cTrader Pro + Copilot, math matches the broker"
