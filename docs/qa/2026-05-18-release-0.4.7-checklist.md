## 📋 Operator-side steps (Windows + VPS)

### 0. Commit-hygiene blockers — MUST clear first

These were flagged by a pre-commit audit. The sandbox cannot run
`cargo` so all four steps run on the Windows host.

```powershell
cd C:\Users\konst\development\forex-ai

# 0.1. Confirm the draft trees are gitignored (already in tree).
#      git status --porcelain should NOT show *_split_draft/ paths.
git status --porcelain | findstr "split_draft"
# Expected: no output. If anything shows, edit .gitignore and re-check.

# 0.2. Refresh Cargo.lock after the 0.4.5 → 0.4.7 version bump.
#      Without this `cargo check --locked` will fail.
cargo update --workspace
# (or `cargo check --workspace` once, which regenerates the lockfile
# transitively without touching transitive-dep versions.)

# 0.3. Repo-wide rustfmt — pre-existing formatting drift, not just
#      this session's changes.
cargo fmt --all
git diff --stat | findstr "^.*|.*\d.*\d.*$"
# Inspect the diff; if any of it looks structural rather than
# whitespace-only, hold and let me know before committing.

# 0.4. Final lock-check + format-check before commit.
cargo check --locked --workspace --all-targets
cargo fmt --all -- --check
```

If any step fails, that's a real blocker — stop and report.

### 3. Build release artifacts on VPS (10-20 min)

```powershell
.\scripts\release-on-vps.ps1
```

This orchestrator (already in tree) does:
1. SSH into Hyperstack VM, syncs the master branch.
2. Runs `cargo build --release` on Linux for the Linux artifact.
3. Optionally cross-compiles Windows artifact (or you run it locally).
4. SCPs the artifact tarball back to your machine.
5. Stops the VM (billing).
