# NeoEthos desktop — gap analysis (2026-06-23)

Audited by driving the running app's in-process API (curl) + reading the code.
Three lenses: **(A) live-data streaming**, **(B) backend features already built
but not surfaced**, **(C) vs professional platforms** (TradingView / MT5 / cTrader).

Account under test: ~99,921 GBP demo (Spotware). Backend port ephemeral (e.g. 51363).

---

## A. LIVE DATA STREAMING — the headline gap

The backend is built for **push**; the UI only **polls**. Verified:

| What | Backend has | UI does | Result |
|---|---|---|---|
| Quotes | `GET /live/spots/stream` (SSE, `freshnessSeconds: 0.0`) | polls `GET /live/spots` every 1.5–2 s | ticks lag ~1–2 s instead of instant |
| Account/equity | `GET /account/snapshot/stream` (SSE, pushed) | polls every 5 s (Tauri cmd) | balance/PnL lag up to 5 s |
| Force refresh | `POST /account/snapshot/refresh` | unused | no instant refresh after an order |
| Chart | `GET /chart` + `GET /chart/history` (scroll-back) | loads **once** per symbol/TF | **last candle never ticks**; no pan-back |

Concrete code: every screen uses `usePoll`/`setInterval` (`desktop/src/hooks.ts`,
`Markets.tsx:102`, `App.tsx:79`, `Dashboard.tsx:34`, `Positions.tsx:34`).
`grep EventSource` → **0 hits**. The SSE endpoints are never opened.

**Fixes (use what already exists):**
1. Replace spot polling with an `EventSource` on `/live/spots/stream` → sub-second ticker + Market Watch.
2. Replace account polling with `/account/snapshot/stream` → live balance/equity/PnL on Dashboard + Positions; `POST /account/snapshot/refresh` right after place/close.
3. **Live chart:** load history via `/chart`, then on each tick update the last candle (O/H/L/C) and roll a new candle at TF boundary — the standard TradingView model.
4. Stream **reconnect/backoff** + a **connection-health pill** (we already receive `freshnessSeconds` / `brokerTimestampMs` — surface staleness).
5. Chart **scroll-back pagination** via `/chart/history` (older bars on pan).

## B. BACKEND FEATURES ALREADY BUILT, NOT IN THE UI

Endpoints that exist in `crates/neoethos-app/src/server/mod.rs` and are **never called** by the frontend (`grep` in `desktop/src` = 0):

**Trading / account**
- `POST /positions/protection` — move SL/TP, breakeven, trailing stop. **Flagship** (per project memory) — completely unused. Pro apps let you drag SL/TP on the chart.
- `GET /broker/margin/expected` — required margin / pip value preview before an order. The order ticket shows none.
- `POST /orders/cancel` — no pending (limit/stop) order management UI at all.
- `GET /broker/orders/history`, `GET /broker/cashflow`, `GET /broker/profile`, `GET /broker/version` — no history/cashflow/profile views.
- `GET /actions/pending` + `POST /actions/{id}/confirm|reject` — the trade-confirmation queue is unused.

**Analysis / config**
- `GET /indicators` — chart has **no indicator overlays** (EMA/RSI/MACD/ATR available, unused).
- `GET /settings`, `GET/POST /settings/raw`, `GET /settings/knob-catalog`, `GET /settings/presets` — the 200+ config knobs + Advanced Settings panel are not editable in the UI.
- `GET /risky/scenarios` — wrapped in `api.ts` but **no screen** renders the growth/time-to-target projection.
- `POST /diagnostics/report` — no "export diagnostics" action.
- `GET /broker/timeframes` — broker TF list hardcoded in `Markets.tsx` instead.

**Architecture note:** Dashboard/Markets/Positions/Settings still use the older
**Tauri commands** (`broker_status`, `account_snapshot`, `broker_chart`,
`spot_prices`) while the 12 new screens use the **HTTP server**. Two parallel
account-snapshot paths. Should unify on the in-process server (it has the SSE
streams the Tauri path lacks).

## C. vs PROFESSIONAL PLATFORMS (needs NEW backend too)

Not in the backend yet — real net-new work:

- **Depth of Market / Level II** order book (cTrader/MT5 staple).
- **Chart drawing tools** (trendlines, fib, rectangles) + saved layouts + multi-pane.
- **Position / order lines drawn on the chart** + one-click trade from chart.
- **Trade markers** (entry/exit arrows) overlaid on history.
- **Price & indicator alerts** + OS notifications.
- **Economic calendar** (we have a news feed, not a dated calendar).
- **Equity-curve & drawdown chart** (journal computes the stats — `maxDrawdownPct`,
  `recoveryFactor`, `sharpe` — but nothing plots the curve).
- **Watchlist columns**: spread, daily %change, sparkline (currently just bid/ask).
- **Multi-account / quick account switch** from the top bar (picker exists in Settings only).

## Priority

1. **Live streaming (A1–A3)** — biggest perceived gap, all backend already exists. Do first.
2. **SL/TP management + margin preview (`/positions/protection`, `/broker/margin/expected`)** — flagship trading capability, backend ready.
3. **Indicators overlay + equity-curve chart** — analysis depth, backend mostly ready.
4. **Advanced Settings (knob-catalog)** — exposes the engine's real power.
5. Net-new pro features (DOM, drawing tools, alerts, calendar) — larger, schedule later.
