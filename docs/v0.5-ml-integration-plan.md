# v0.5 — ML ensemble integration plan (overfit ↓ + resurrect dead voters + connect genes↔ML + re-validate OOS)

**Status:** DESIGN COMPLETE + adversarially verified (2026-06-07). Not yet implemented.
**Operator directive:** "Μείωση του overfit, σύνδεση με τις στρατηγικές, σύνδεση των τώρα νεκρών μοντέλων στην ψήφο, γενικά δεν πετάμε αξιοποιούμε, συνδέουμε πραγματικά ότι έχουμε."

## The hard constraint (why this is staged, not a big-bang merge)

The **genes are OOS-validated** (faithful forward test: 32 survivors, PF 2–3.5, DD ≤ ~8%, WFE ≥ 0.5).
The **ML ensemble is NOT validated** (trains on raw per-(symbol,TF) bars + triple-barrier labels; never wired to production; overfit-prone defaults; never re-validated OOS).

Therefore "connect everything" must be **structurally incapable of degrading the validated gene edge**:
- ML **never** sets DIRECTION — genes are the sole direction source. ML only **scales confidence/size** in `[gate_floor,1]` and may **veto-to-Flat**. It can never flip Long↔Short and never turn Flat into a trade (López de Prado meta-labeling: primary=genes→direction, secondary=ML→bet/size).
- **Default OFF / byte-identical fallback:** `blend_mode=GenesOnly` (or ML absent / load-fail / column-mismatch) → output is byte-identical to today's gene-only `PrecomputedSignalEngine`. Hard invariant + test.
- **Accept-only-if-beats-genes:** the blend ships ON for a (symbol,TF) ONLY if it Pareto-dominates genes-alone on the locked 2023+ holdout, measured through the **same faithful engine on the same bars/costs**. Any failure → REJECT → production stays genes-alone.

## Stage 1 — Overfit reduction (foundational, zero touch to genes)

Three ML-training fixes, all inside `neoethos-models` + one shared-CV hoist to core.

### (a) Regularized, bar-scaled booster defaults
- **XGBoost:** max_depth 8→4, learning_rate 0.05→0.03, subsample 1.0→0.8, colsample_bytree/bylevel/bynode 1.0→0.8, +min_child_weight=10, +reg_lambda=5.0, +reg_alpha=0.5, +gamma=0.5; n_estimators scaled (bars<3000→200, 3000–10000→400, >10000→800) with early-stop on the existing val frame.
- **LightGBM:** num_leaves 31→15, max_depth 8→4, min_data_in_leaf 1→max(20,bars/200), min_data_in_bin 1→3, feature_fraction/bagging_fraction 1.0→0.8, bagging_freq 0→1, min_gain_to_split 0→0.01, lambda_l1 0→0.5, lambda_l2 0→5.0; num_iterations scaled 200/400/800.
- **CatBoost:** depth 8→4, l2_leaf_reg 3→6, +random_strength=1.0, +bagging_temperature=1.0, subsample/rsm=0.8; iterations scaled 200/400/800.

> **⚠️ ADVERSARIAL CORRECTION (verdict #3 — load-bearing):** editing the three `default_params()` functions ALONE is a **silent no-op** on the production path. `XGBoostExpert::new`/`LightGBMExpert::new`/CatBoost do `params.unwrap_or_else(Self::default_params)`, but the orchestrator ALWAYS passes `Some(parse_tree_params(...))` (build_expert_model:2590/2604/2616), and `parse_tree_params` (1990-2010) does NOT merge `default_params()` — it only type-converts the keys the orchestrator set. So every regularization key must be written in **TWO** real places:
> 1. The **orchestrator seed maps** `default_model_params` (training_orchestrator.rs:1009-1064) — the real production source for named boosters.
> 2. The **inline getter fallbacks** in each booster train path (xgboost.rs:194/204/214/658/685; lightgbm.rs:255-268; catboost.rs:407/690) — so an unset key defaults to the regularized value, not the unregularized one.
> Editing `default_params()` is optional cosmetic (None/test path only).
> **Add a unit assertion** that a default-config-built `ModelConfig` for xgboost/lightgbm/catboost actually contains the reg keys (e.g. subsample≤0.8, lambda_l2>0) — prevents regression back to the silent-no-op.

### (b) Per-(symbol,TF) min-bar gate
In `train_model_dispatch` (training_orchestrator.rs:3370): `bars = payload.frame.height()`.
- `bars < heavy_booster_min_bars` (new config, default **4000**; D1 ~2700 falls below) for heavy boosters → force shrunk preset (depth 3, n_estimators 150, ≤7 leaves, strong L2) AND force trials=1 (a thin holdout cannot select 5+ hyperparameters).
- `bars < 800` (absolute floor) → skip the booster, record a skipped artifact (it simply does not vote).
- **Assert ≥1 valid Classification3 voter survives per (symbol,TF), or fail loud.**

### (c) CPCV for ML model selection (replace the single holdout)
`optimize_model_config` (training_orchestrator.rs:3169-3213) today does ONE `time_series_holdout_split` and picks best_params on that one slice — the HPO sampler overfits it.
- **HOIST `CombinatorialPurgedCV`** from neoethos-search/src/validation.rs:1224-1326 **into neoethos-core** (single impl shared by genes+ML — matches the SystemConfig single-source pattern; avoids a models→search crate cycle).
- Replace the single split with `CombinatorialPurgedCV::new(6,2,0.02,0.01).split(...)` (15 purged paths). Score each candidate by **MEAN-MINUS-STDEV** of objective_score across paths (penalize variance = the direct overfit signal). Keep the full-data refit for deployment.
- **Gate behind** `bars≥heavy_booster_min_bars AND trials>1` to bound cost (reuse hpo_max_rows budget + a cpcv_max_rows cap); else fall back to single holdout / trials=1.

**Flags (ModelsConfig/DiscoveryRuntimeConfig, default ON, env-overridable):** `regularized_defaults`, `heavy_booster_min_bars=4000`, `ml_cpcv_enabled`. Validate by **OOS (faithful_oos_eval)**, never by training accuracy (in-sample accuracy is EXPECTED to drop).

## Stage 2 — Re-role the "dead" voters (self-contained in neoethos-models)

**Correction to the brief (verified):** the dead voters are NOT discarded by output-kind — almost every model already reports `Classification3` and therefore already votes, but with WRONG/destructive semantics in the flat average. The real fix is **re-roling**, not resurrection. Replace the flat-average `predict` (soft_voting.rs:200-280) with a role-aware combiner emitting `EnsembleDecision { dir_probs:[f32;3], regime_gate:f32, anomaly_scale:f32 }`:

- **hmm_regime → REGIME GATE** (not a direction voter). Posterior col0=P(range), col1=P(buy), col2=P(sell) (hmm_regime.rs:524-525). `g = (1 - p_range) * directional_agreement` ∈[0,1]. Range→g≈0 (veto/shrink); agreeing trend→g≈1; disagreeing trend→g small. Degrade to g=1 on first-row uniform prior / insufficient data (never veto-by-accident).
- **isolation_forest → ANOMALY SCALE/VETO** (not a direction voter). Raw score `a = col0` is **already in the existing Classification3 output** (forest_impl.rs:415-420) — **no retrain / no new artifact needed** (verdict #2 refinement). `anomaly_scale = clamp(1 - max(0,(a-a_lo)/(a_hi-a_lo)),0,1)`, a_lo/a_hi operator knobs (default 0.5/0.9). Hard veto at `a≥a_hi`. Degrade to s=1 on missing columns.
- **dqn → CONFIRM-ONLY directional contributor.** Already softmaxes Q-values into Classification3 (rl_exit_adapters.rs:146-158). Keep in the directional pool as confirm-only (can add weight to the gene's side, never override). *(Fix the STALE doc at mod.rs:166-172 that claims argmax.)*
- **Forecast1 deep-TS (nbeats/tide/patchtst/timesnet):** already 3-class — do NOT double-handle. Only the genuine continuous `swarm_forecaster` (deferred) needs a sign-threshold shim when un-deferred.
- **exit_agent / ExitDecision3 → DEFERRED.** Removed in F-318, not bootstrapped. The `ExitAdvisor` trait seam + engine.rs:189 position-management consumption is a future ADD, not part of this stage.

**DIRECTION vote** = weighted average of ONLY genuine directional classifiers (trees, deep-cls, deep-TS-3class, statistical, online_*, meta_*, dqn-confirm) via `expert_weights` (default 1.0).
**Role partition (verdict #2 refinement):** use an **explicit name→role map** declared in one place (mod.rs), **fail loud on an unmapped expert** (don't partition on coarse `ModelFamily` — hmm_regime is Meta, same family as the directional meta_blender). Unit-test DEFAULT_BOOTSTRAP_EXPERT_NAMES against the map. Assert ≥1 directional voter survives.

## Stage 3 — Connect genes↔ML in the production signal (gene-dominant meta-gate, default OFF)

New `crates/neoethos-trader/src/blend_signal.rs`: `BlendedSignalEngine impl SignalEngine` (engine.rs loop untouched, parity preserved). Holds `genes: Vec<Direction>` (the SAME `combine_gene_signals` net, reused verbatim), `ml: Option<Vec<EnsembleDecision>>`, `BlendConfig { mode, gate_floor, veto_below }`, per-symbol cursor.

Per bar (gene direction d, `EnsembleDecision{dir_probs, g, s}`):
```
p_side = dir_probs[buy] if d==Long else dir_probs[sell]   // Flat → 0.0, no trade
m      = clamp(p_side * g * s, gate_floor, 1.0)
MlConfirm: dir_out = Flat if p_side < veto_below else d ; conf_out = 0 if Flat else m
MlScale  : conf_out = max(gate_floor, p_side)             // no veto
GenesOnly (DEFAULT): ml=None → conf_out = 1.0/0.0 == byte-identical to today
```
**Invariants (hard, tested):** ML can NEVER flip Long↔Short, NEVER turn Flat into a trade; `gate_floor` (default 0.34) guarantees a good gene bar always trades a meaningful size; `gate_floor=1.0` is a no-op escape hatch; ensemble-absent/load-fail/column-mismatch → floor=1.0 (gene-only). `source = SignalSource::Blend`.

**Prerequisite fix:** `combine_gene_signals` (gene_signal.rs:54-66) discards the per-bar gene confidence that `signals_and_confidence_for_gene_full` already computes (consumed by faithful_oos_eval). Add `combine_gene_signals_with_confidence` that nets/averages gene confidence so the blend scales a REAL `gene_confidence`: `final = gene_conf * m`.

**Wiring (parity, fail-loud):** add `neoethos-models` dep to neoethos-trader/Cargo.toml (optionally behind a `ml-blend` feature). New `replay_blend_from_dir` alongside `replay_portfolio_from_dir` (kept as validated baseline). Build the ML DataFrame from `raw_features` (NOT the gene-projected `aligned`) via the `from_named_dense` recipe; `build_ensemble_for_symbol`; **read `feature_columns()` off the loaded experts, assert all agree, select the df to exactly that set; on ANY mismatch → FAIL LOUD → GenesOnly** (experts bail unless df columns == expected exactly). Neutral-fill warmup/NaN bars ([1,1,1]/3 → m≈gate_floor, ML abstains). Assert `ml.len()==aligned.n_samples()==base_ohlcv.len()`. CLI `trader-replay` + `/autonomous/replay` get `--blend off|confirm|scale` + `--models-root` through the SAME EngineConfig (UI↔CLI parity mandate).

## Stage 4 — Re-validate the blend OOS (the acceptance gate; built early, runs continuously)

1. **LEAKAGE FIX (mandatory first).** On-disk experts have SEEN 2023+ (training uses only a fractional most-recent holdout). Add `oos_from_ms: Option<i64>` to `train_symbol_with_progress` (orchestrator:239-256) that truncates frame+labels to `timestamp < oos_from` minus a `hold_bars` purge (triple-barrier looks forward), BEFORE the split, writing to a SEPARATE root `models_oos_locked/<sym>/<tf>/` (production `models/` untouched). The test REFUSES the production root and fails loud.
2. **Lock one cutoff:** `--oos-from 2023-01-01` (already default). IS=(-inf,cut), OOS=[cut,end). Tail never trained/HPO'd/feature-selected/tuned on.
3. **Inject the SAME gate into faithful_oos_eval.** Add `faithful_oos_eval_blended` + a pure unit-testable `apply_ml_gate(sig, conf, decisions, cfg)` that runs AFTER `signals_and_confidence_for_gene_full` and BEFORE `fast_evaluate_strategy_core` — same bars, same SL/TP, same costs → strictly apples-to-apples. Load `models_oos_locked`, rebuild the cube warm over the full series, slice to [oos_start,end). Persist higher_tfs/options/feature_columns in the OOS-expert artifact; by-name validate, fail loud on drift.

> **⚠️ ADVERSARIAL CORRECTION (verdict #1 — critical):** the live trader **NETS** genes (`combine_gene_signals` → one signed per-bar signal), but the naive Stage-4 gate validates **each gene alone**. Netting is non-linear → a blend that beats per-gene can still degrade the netted live portfolio. **The accept-gate MUST validate the NETTED portfolio the live engine actually trades:** net per-gene (signal+confidence) over the holdout via `combine_gene_signals_with_confidence`, run the backtest core ONCE on the netted signal, apply the gate AFTER netting (mirroring `BlendedSignalEngine`). Add a parity test that the netted holdout run == the live `replay_blend_from_dir` with the gate off. Also: `min_volume=0.01` means scale-only can't skip a trade, so a "skip" must set direction to Flat given the volume floor.

4. **Measure** both arms on the IDENTICAL 2023+ slice via GeneOosResult/BacktestMetrics: OOS PF, max_drawdown, net, trades, Sharpe, WFE_sharpe — at **portfolio (netted) level** AND per-gene. Paired delta table.
5. **ACCEPT/REJECT (gene edge sacred).** Promote blend ON for a (symbol,TF) ONLY if ALL hold on the NETTED holdout: (1) blend OOS PF ≥ genes-alone PF; (2) blend OOS max_drawdown ≤ genes-alone DD; (3) blend OOS net>0 AND trades≥30; (4) blend WFE_sharpe ≥ genes-alone WFE_sharpe. Any failure → REJECT → production stays genes-alone. Fail-safe by default.
6. **Keep the holdout locked.** gate_floor/veto_below/tau/voter toggles tuned on **IS ONLY** (inner IS-CPCV or IS walk-forward); the 2023+ slice evaluated EXACTLY ONCE per frozen config; log a "holdout used N times" counter to the forward-test artifact ledger. CLI: extend `cmd_forward_test` (or a `cmd_blend_test`) to run BOTH arms + print the delta table + verdict.
